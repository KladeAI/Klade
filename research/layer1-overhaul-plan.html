The 7-Day Plan

Execution rules: All ingestion runs as background subagents -- never on main session. Dev manages orchestration + quality checks. If any pipeline breaks, Dev auto-fixes and continues (locked rule). Arjun involvement: zero, unless a paid service or external action is needed. Estimated daily compute cost: ~$2-5 in OpenAI embedding API calls.
DAY 1 -- Thursday March 27: XBRL Bulk Download + S&P 500 Completion

Morning: S&P 500 10-K Finish

  • Complete remaining ~65 companies (ingestion already running)
  • Verify all 500 companies have at least one 10-K indexed
  • Fix any failed tickers from prior batches

Afternoon: XBRL Pipeline Build + First Ingest

  • Download companyfacts.zip (~multi-GB, one file, all companies)
  • Build XBRL ingestion script: parse JSON, extract key financial metrics per company per period
  • Target metrics: Revenue, Net Income, EPS, Total Assets, Total Liabilities, Stockholders' Equity, Operating Cash Flow, CapEx, Shares Outstanding
  • Create new Supabase table: xbrl_facts (company_id, metric, period, value, unit, source_filing)
  • Ingest all S&P 500 XBRL data (all available periods -- typically 5-10 years per company)

End of Day 1 deliverable: 500/500 S&P companies with 10-K text + XBRL structured financials for all available periods.

Estimated records added: ~2M XBRL data points

DAY 2 -- Friday March 28: 10-Q Quarterly Filings (Batch 1)

Full Day: 10-Q Ingestion Pipeline

  • Modify existing ingestion script to handle 10-Q filings
  • Target: last 8 quarters (2 years) of 10-Qs per company
  • Run in batches of 50 companies (10 batches total)
  • Rate limiting: 8 req/sec to SEC, with exponential backoff on 429s
  • Batch 1-5 (250 companies): Run during day

End of Day 2 deliverable: 250 companies with quarterly filings ingested (~1,000 10-Qs)

Estimated records added: ~150K filing chunks

DAY 3 -- Saturday March 29: 10-Q Completion + 8-K Material Events
